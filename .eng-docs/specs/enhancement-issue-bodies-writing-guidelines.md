# Enhancement: Issue bodies follow writing guidelines

## Parent feature

`feature-github-issue-triage.md`

## What

The triage skill applies the `caf:writing-guidelines` style when generating issue titles and bodies. This includes sentence case for all headings, active voice, specific and concrete language, and no unnecessary jargon. The skill loads or references the writing guidelines before generating any issue content.

## Why

Issue content generated by the triage skill currently does not follow the same writing standards applied to other CAF artifacts. Inconsistent style makes the backlog harder to read and undermines the value of having guidelines at all. Applying them consistently means every issue is as clear and readable as the planning artifacts that reference it.

## User stories

- Devon can read any issue created by the triage skill and find the same writing quality as other CAF artifacts
- Petra can scan the backlog without encountering title-case headings or passive voice in triage-generated issues
- Devon can rely on the triage skill to apply writing guidelines without reviewing each issue for style
